# 배포 가이드

이 프로젝트는 현재 Codex에서 로컬 STDIO MCP 서버로 실행하는 방식을 기준으로 관리합니다. 외부 서버로 공개 배포하거나 원격에서 네이버 계정을 대신 조작하는 형태는 권장하지 않습니다.

## 권장 실행 방식

1. 이 저장소를 로컬에 둡니다.
2. 저장소 루트의 `.env`에 네이버 계정 정보를 저장합니다.
3. `~/.codex/config.toml`에는 비밀번호 없이 MCP 실행 명령만 등록합니다.
4. Codex가 필요할 때 `uv run naver-blog-mcp`를 STDIO로 실행하게 둡니다.

자세한 설정은 [`codex-guide.md`](codex-guide.md)를 보세요.

## 운영 시 주의사항

- `.env`와 `playwright-state/`는 절대 커밋하지 않습니다.
- `SAVE_DEBUG_ARTIFACTS=false`를 기본값으로 둡니다.
- 이미지 업로드는 `blog-images/` 폴더 안 파일만 쓰는 기본 정책을 유지합니다.
- 실제 공개 발행 전에 `publish=false`로 임시저장 테스트를 먼저 합니다.

## 외부 배포를 피하는 이유

네이버 블로그 자동화는 사용자의 실제 브라우저 세션과 계정 보안 정책에 영향을 받습니다. 원격 서버로 배포하면 비밀번호, 세션 쿠키, 작성 중인 글 내용, 디버그 스크린샷이 서버에 남을 수 있습니다. 따라서 현재 형태는 개인 로컬 환경에서만 쓰는 것이 안전합니다.
